Datenstrukturen — die fünf, die zählen

R hat nur eine Handvoll Strukturen. Wer sie durchschaut, liest jeden Code. Die data.frame ist dein täglicher Arbeitsplatz, der factor deine häufigste Fehlerquelle.

Warum diese Lektion

In Lektion 1 hast du Vektoren als Grundelement von R kennengelernt. Jetzt erweitern wir das Spektrum: matrix, data.frame, list und factor kommen dazu. Für deine spätere PMM-Arbeit wirst du fast nur mit data.frame arbeiten — aber factor brauchst du für jede ANOVA und jeden χ²-Test, und list triffst du in jedem Modell-Ergebnis an (aov(), lm()). Die Investition lohnt sich.

Die fünf Strukturen auf einen Blick

StrukturDimensionTypWofür
vector1Dgleichartig (homogen)Messreihe, Spalte
matrix2Dhomogenrein numerische Matrizen, Lineare Algebra
data.frame2DSpalten je ein TypTabellen — dein Alltag
list1DbeliebigModell-Ergebnisse, verschachtelte Daten
factor1Dkategorial (intern integer)Gruppen, Nominal-/Ordinalskalen
Die Regel, die alles erklärt

Homogen (vector, matrix) erzwingt einen Typ — mischst du, zwängt R alles in den „gemeinsamen Nenner". Heterogen (data.frame, list) erlaubt gemischte Typen, weil jedes Element/eine Spalte eigener Typ sein darf.

Vorab: Typ-Erzwingung (Coercion)

Wenn du unterschiedliche Typen in c() steckst, entscheidet R über eine feste Hierarchie, welcher Typ „gewinnt":

c(1, "a")        # logical < integer < double < character
c(1, TRUE)      # TRUE wird zu 1
c(TRUE, "a")
Erst vorhersagen, dann zeigen 
[1] "1" "a"
[1] 1 1
[1] "TRUE" "a"

Alles wird zum „höchsten" Typ in der Hierarchie. "a" ist character → alles wird character; 1 wird zu "1".

Vorhersage
Welchen Typ hat c(1, "a", TRUE) nach der Coercion?
  • Zeichenkette am Ende
  • Ganzzahl bleibt bestehen
  • Wahrheitswert bleibt
  • Fehler wird geworfen
Richtig. character steht in der Hierarchie ganz oben — alles wird zu Text.
Falsch. Die Hierarchie ist logical < integer < double < character. Da "a" dabei ist, wird alles zu character.

1. Vector — das Grundelement (Rekap)

Bekannt aus Lektion 1: gleichartig, 1D, gebaut mit c(). Vier atomare Typen: logical, integer, double, character.

x <- c(8.1, 7.9, 8.0)
class(x)     # "numeric" (double)
length(x)    # 3
Output 
[1] "numeric"
[1] 3

2. Matrix — 2D, aber homogen

Eine Matrix ist im Grunde ein Vektor mit einem dim-Attribut. Weil sie homogen ist, passt nur ein Typ — mischst du Zahlen und Text, wird alles Text. In der Statistik eher für reine Zahlenmatrizen.

matrix(1:6, nrow = 2, ncol = 3)          # füllt SPALTENWEISE
matrix(1:6, nrow = 2, ncol = 3, byrow = TRUE)
Output 
     [,1] [,2] [,3]
[1,]    1    3    5
[2,]    2    4    6
     [,1] [,2] [,3]
[1,]    1    2    3
[2,]    4    5    6

Achtung: default ist spaltenweise — das überrascht Python-Nutzer, die an zeilenweise Füllung gewöhnt sind.

3. data.frame — dein täglicher Arbeitsplatz

Die data.frame ist eine Tabelle: Zeilen = Beobachtungen, Spalten = Variablen. Jede Spalte ist ein eigener Vektor mit eigenem Typ. Damit ist sie heterogen — und genau das willst du für reale Daten (Messwert = numeric, Maschine = factor, Datum = Date, …).

df <- data.frame(
  id      = 1:3,
  messung = c(8.1, 7.9, 8.2),
  maschine = c("A", "A", "B")
)
df
str(df)         # die wichtigste Inspektionsfunktion
Output 
  id messung maschine
1  1     8.1        A
2  2     7.9        A
3  3     8.2        B
'data.frame':	3 obs. of  3 variables:
 $ id     : int  1 2 3
 $ messung: num  8.1 7.9 8.2
 $ maschine: chr  "A" "A" "B"
str() ist dein bester Freund

str(df) zeigt Struktur, Typ jeder Spalte und erste Werte. Mache das immer als erstes, wenn du Daten lädst — noch vor head() oder summary().

Seit R 4.0

Früher wurden Textspalten in data.frame automatisch zu factor. Seit R 4.0 sind sie character. Schüler mit alten Lehrbüchern wundern sich — kläre diese Historie, wenn du factor behandelst.

Verständnis
Welche Struktur hält Spalten mit unterschiedlichem Typ?
  • data.frame genügt
  • matrix genügt auch
  • vector reicht aus
  • factor passt immer
Richtig. Eine data.frame erlaubt pro Spalte einen eigenen Typ — genau deshalb ist sie der Standard für Tabellen.
Falsch. matrix ist homogen (ein Typ für alles); nur data.frame erlaubt gemischte Spaltentypen.

4. list — der Allzweck-Container

Eine list ist wie ein Vektor, aber jedes Element darf ein anderer Typ und eine andere Struktur sein — sogar weitere Listen. Darin liegen Modell-Ergebnisse, API-Antworten, komplexe Objekte.

modell <- list(
  koeffizient = c(7.8, 0.3),
  p_wert      = 0.012,
  daten       = df
)
modell$koeffizient       # Zugriff per Namen ($)
modell[[1]]              # Zugriff per Position
modell[1]                # Einfach vs. doppelt — was ist anders?
Erst vorhersagen, dann zeigen 
[1] 7.8 0.3
[1] 7.8 0.3
$koeffizient
[1] 7.8 0.3

Optisch sehen modell[1] und modell[[1]] auf den ersten Blick gleich aus — aber der Typ des Ergebnisses ist verschieden:

class(modell[1])     # "list"     <- eine Liste bleibt eine Liste
class(modell[[1]])   # "numeric"  <- Inhalt ist der Vektor selbst
length(modell[1])    # 1          <- ein Listenelement
length(modell[[1]])  # 2          <- der Vektor hat zwei Werte
Output 
[1] "list"
[1] "numeric"
[1] 1
[1] 2
Deine Metapher: „Im Fach liegt eine Box"

Stell dir modell als Regal mit Fächern vor. In jedem Fach liegt eine Box, und in der Box steckt der eigentliche Inhalt (Vektor, Skalar, …):

SchreibweiseSpracheErgebnis
modell[1]„Gib mir die Box aus Fach 1"Liste (die Box) — Inhalt noch verpackt
modell[[1]]„Öffne die Box aus Fach 1, gib mir den Inhalt"nackter Wert (numeric, …)
modell$koeffizientKurzform für [[ per NamenInhalt, identisch mit modell[[1]]

Die doppelte Klammer greift eine Ebene tiefer — sie durchstößt die Boxwand. Darum: [ = Form behalten, [[ = Inhalt holen. Ein schöner Beweis:

identical(modell[1][[1]], modell[[1]])   # TRUE

Erst die Box ([1]), dann öffnen ([[1]]) — identisch mit direktem Öffnen. [ ist darum auch die häufigste Fehlerquelle für mean() auf Listenelementen: mean(modell[1]) → Fehler, mean(modell[[1]]) → 4.05.

5. factor — kategorial, und die große Falle

Ein factor speichert kategoriale Werte: äußerlich sieht er aus wie Text, intern ist er ein Integer-Vektor mit Labels (den „levels"). Für Statistik unverzichtbar — ANOVA, χ²-Test und Regelkarten brauchen factor für Gruppierungen.

quali <- factor(c("niedrig", "mittel", "hoch", "mittel"))
quali
levels(quali)
as.numeric(quali)        # ACHTUNG — siehst du die Falle?
Output 
[1] niedrig mittel  hoch    mittel
Levels: hoch mittel niedrig
[1] "hoch" "mittel" "niedrig"
[1] 3 2 1 2

Die Levels werden alphabetisch geordnet: hoch=1, mittel=2, niedrig=3. Also ist "niedrig" intern die 3. as.numeric(quali) liefert 3 2 1 2 — die Codes, nicht die Werte.

Die klassische Falle

Hast du Zahlen als Text gespeichert und wandelst sie über factor zurück, bekommst du nicht die Zahlen:

as.numeric(factor(c("10", "20", "30")))   # NICHT 10 20 30

Ergebnis: 1 2 3 — die Factor-Codes. Korrekt: as.numeric(as.character(...)).

Die Falle
Was liefert as.numeric(factor(c("10","20","30")))?
  • Eins zwei drei kommt
  • Zehn zwanzig dreissig
  • Fehler wird geworfen
  • Text bleibt erhalten
Richtig. factor legt Codes 1, 2, 3 an (alphabetisch sortierte Levels) — und as.numeric() holt genau diese Codes, nicht die Werte.
Falsch. Der Factor speichert intern Integer-Codes. as.numeric() gibt die Codes 1 2 3, nicht 10 20 30. Richtig wäre as.numeric(as.character(...)).

Ordnung mitbringen: ordered factor

Nominalskalen (A, B, C) sind unsortiert. Ordinalskalen (niedrig < mittel < hoch) brauchen eine Reihenfolge:

quali_o <- factor(c("niedrig","mittel","hoch"),
                  levels = c("niedrig","mittel","hoch"),
                  ordered = TRUE)
quali_o
quali_o[1] < quali_o[2]   # TRUE — Vergleich möglich
Output 
[1] niedrig mittel  hoch
Levels: niedrig < mittel < hoch
[1] TRUE

Didaktik — Schülern diesen Stoff erklären

Drei Lehr-Entscheidungen
  1. factor als eigene kleine Einheit. Zeige zuerst den Integer-unter-der-Haube-Effekt (as.numeric(factor("10")) → 1), bevor du ANOVA einführst. Sonst verwechseln Schüler später Factor-Codes mit Messwerten.
  2. matrix vs. data.frame kontrastieren. Ein einziger Versuch — matrix(c(1,"a")) wird zu character — prägt sich tiefer ein als jede Erklärung.
  3. str() zur Pflicht machen. „Erst str(), dann alles andere" als Unterrichtsroutine einführen. Das verhindert 80 % der Folgefehler.
Prüfungsfalle

„Wandle die Factor-Spalte note (Werte „1", „2", „3") in numerisch um." — wer as.numeric(note) schreibt, bekommt 1 2 3 als Codes statt als Noten. Klassische Aufgabenstellung; Schüler müssen as.character dazwischen schalten.

Hausübung

  1. Lade mtcars und führe str(mtcars) aus. Welche Spalten sind numeric, welche könnten sinnvoll als factor sein (Tipp: cyl, gear)?
  2. Wandle mtcars$cyl in einen Factor um. Was liefert levels()?
  3. Vorhersage (ohne R): as.numeric(factor(c("5","5","10"))) — dann prüfen und erklären.
  4. Baue eine data.frame mit einer numeric-, einer character- und einer factor-Spalte. Rufe summary() auf und vergleiche, wie die drei Spalten unterschiedlich zusammengefasst werden.
  5. Baue eine list, die einen Vektor, eine data.frame und einen Skalar enthält. Greife jedes Element einmal per $ und einmal per [[ ]] zu.
